# Examples

Three end-to-end verified outputs from the toolkit. Open the source files
to inspect; rendered previews are included where applicable.

## 1. Transformer encoder-decoder (`transformer.drawio`)

Generated by adapting `references/drawio-templates.md` §1. Open in
[draw.io](https://app.diagrams.net/) or VSCode's Draw.io extension.

**What this verifies:**
- Flow Direction rule — every forward edge has `source.y > target.y`
- Cross-Stack Y-Alignment — encoder output (enc_an2) and decoder
  cross-attention (dec_ca) share Y-center 435, K,V arrow is a clean
  horizontal line with no waypoints
- No-Overlap rule — section containers have ≥30px clear gap, labels
  inside top-left don't intrude on neighbors
- HTML escapes — `Add &amp; Norm` (single escape, not double)

**To use as a starting point for your own paper figure:** open the file,
rename modules, change colors per `references/drawio-reference.md`'s IEEE
palette, adjust `× N` annotation in the section labels.

## 2. Diffusion forward / reverse process (`diffusion.drawio`)

Generated by adapting `references/drawio-templates.md` §2 (DDPM-style
two-direction Markov chain).

**What this verifies:**
- LR flow direction with the documented exception for the reverse chain
  (`source.x > target.x` allowed because the process flows right-to-left
  by definition)
- Title color matches arrow color (forward blue, reverse orange — no
  legend needed)
- Math captions positioned above the chain row (~20px gap), no overlap with
  reverse chain that follows ~140px below

**To adapt:** replace x_t placeholders with actual sample-image embeds
(via `<img src="data:image/png;base64,...">` HTML in the value attribute,
or drawio's `shape=image` shape) for a true "denoising trajectory" figure.

## 3. SOTA comparison bar chart (`sota-comparison.py` + `.pdf` + `.png`)

Generated by adapting `references/plotting-templates.md` §I-1. The script
is self-contained — `python sota-comparison.py` produces both `.pdf`
(vector) and `.png` (600 dpi).

**What this verifies:**
- `pdf.fonttype = 42` — PDF embeds CIDFontType2 / Type 0 (TrueType
  composite), passing ACM/IEEE submission Type-3 detection
- IEEE single-column figure size (3.5 × 2.6 in)
- IEEE-semantic palette with "Ours" in attention-purple
- Error bars disclosed in caption (`±1 SD over 5 seeds`)
- Y-axis zoom-in to 60-95% so differences are visible (disclosed)

**Caption draft** (paste into your LaTeX `\caption{...}`):

> Performance comparison on four standard benchmarks. Bars show mean
> accuracy over 5 random seeds; error bars indicate ±1 standard deviation.
> Our method (purple) consistently outperforms both baselines across
> MMLU, GSM8K, HumanEval, and BBH. The y-axis starts at 60% to make
> differences visible.

**To adapt:** replace `methods`, `metrics`, `values`, `errs` arrays at
the top of the script. Adjust `ax.set_ylim(60, 95)` to your data range.
